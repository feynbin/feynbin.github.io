# Go初级开发简历技能点构思

> 学了15篇Go系列文章，从变量写到单元测试，现在该把这些知识**翻译成简历语言**了。这篇文章不是教你"怎么写简历"，而是帮你梳理：**你已经掌握了什么，哪些能写上去，怎么写才不虚也不怂。**

---

## 一个核心原则：写上去的，必须能接住追问

简历上每个技能点都是面试官的提问入口。写了"熟悉channel"，面试官就会问"channel底层结构是什么？有缓冲和无缓冲的区别？close之后读写会怎样？"

所以技能点不是越多越好，而是**每一条都能展开说2-3分钟**。写不了的宁可不写，写上去的必须兜得住。

---

## 技能点推荐写法

### 第一条：语言基础（必写）

> 熟悉Go语言，掌握slice、map等核心数据结构及其底层实现，理解值类型与引用类型的区别。

**你能接住的追问：**
- slice扩容策略是什么？（1.18前后的变化）
- map的底层是什么结构？为什么不是线程安全的？
- 数组和切片的区别？
- 值接收者和指针接收者的区别？什么时候用哪个？

### 第二条：并发编程（必写，这是Go的核心差异化）

> 熟悉Go并发编程模型，掌握goroutine、channel、select的使用，了解sync包常用同步原语（Mutex、RWMutex、WaitGroup）。

**你能接住的追问：**
- goroutine和线程的区别？
- 有缓冲channel和无缓冲channel的区别？
- 向已关闭的channel发送/读取会怎样？
- Mutex和RWMutex的区别？什么场景用RWMutex？
- sync.Map和普通map+Mutex的区别？

### 第三条：错误处理（推荐写）

> 理解Go的错误处理机制，掌握error接口、错误包装（%w）与解包（errors.Is/As），了解panic/recover的使用场景与约束。

**你能接住的追问：**
- Go为什么没有try/catch？
- %w和%v包装错误的区别？
- panic和error分别什么时候用？
- recover有哪些限制？（必须在defer中、不能跨goroutine）

### 第四条：接口与类型系统（推荐写）

> 理解Go的接口机制与隐式实现，掌握类型断言、type switch、空接口的使用。

**你能接住的追问：**
- Go的接口是怎么实现的？和Java的接口有什么区别？
- 什么是"鸭子类型"？
- 空接口 `interface{}` / `any` 有什么问题？

### 第五条：测试（推荐写——体现工程素养，很多初级开发不写测试）

> 掌握Go标准测试框架，能编写表驱动测试和子测试，了解TestMain生命周期管理和覆盖率分析。

**你能接住的追问：**
- 测试文件和函数的命名约定？
- 什么是表驱动测试？为什么推荐？
- TestMain的作用和执行流程？
- t.Errorf和t.Fatalf的区别？

### 第六条：泛型（可选加分项）

> 了解Go泛型（Go 1.18+），能使用类型参数编写通用函数和数据结构。

**注意：** 泛型是加分项但不是必须。初级岗位面试官可能不会深问，但写上说明你跟进了语言新特性。如果对泛型的约束（constraints）和底层实现说不太清楚，可以不写。

## 几个容易踩的坑

### 不要写"精通"

初级开发写"精通Go"，面试官会直接拉满难度。用**"熟悉"或"掌握"**，给自己留空间。

```
❌ 精通Go语言及其并发编程模型
✅ 熟悉Go语言，掌握并发编程基础
```

### 不要罗列关键词

一堆技术名词堆砌在一起，面试官看不出深度。

```
❌ Go / goroutine / channel / mutex / interface / defer / panic / recover / slice / map
✅ 熟悉Go并发编程，掌握goroutine、channel、select及sync包同步原语的使用
```

### 不要写你不能展开的东西

如果你对GMP调度模型、GC三色标记还没有深入学习，不要写"了解Go运行时调度机制"。面试官看到这句话一定会问GMP，答不上来比没写更扣分。

```
❌ 了解Go运行时原理，包括GMP调度、GC机制、内存逃逸分析
  （如果你还没深入学这些，不要写）
✅ 等你真正学了再加上去
```

### 技能点要和项目经验呼应

简历上写了"掌握goroutine和channel"，项目经验里最好有用到并发的地方。技能点和项目经验互相印证，可信度翻倍。

```
技能点：熟悉Go并发编程，掌握goroutine、channel...
项目经验：...使用goroutine+channel实现消息的异步广播和持久化...
```

## 你目前还缺什么

坦诚地说，15篇文章覆盖了Go语言本身，但一个Go后端开发的简历还需要：

| 维度 | 你有的 | 还需要的 |
|------|--------|---------|
| Go语言 | 语法、并发、错误处理、测试 | GMP调度、GC原理、内存逃逸（底层原理） |
| Web框架 | — | Gin/Echo（至少一个，会CRUD+中间件） |
| 数据库 | — | MySQL（索引、事务、EXPLAIN）、GORM |
| 缓存 | — | Redis（五大类型、持久化、穿透/雪崩） |
| 网络 | — | HTTP协议、TCP基础、WebSocket |
| 工具链 | — | Git、Docker基础、Linux常用命令 |
| 项目 | — | 至少一个能说清楚的Go项目 |

> 这不是说你现在的简历不能投——参考你之前写的[《求职冲刺五天计划》](/posts/ac41d369/)，**边学边面，边面边补**。先把Go语言这一栏写好投出去，其他的边学边加。
